Data Preprocess

Data Cleaning

We performed data cleaning on the dataset of global media reports on the China–US trade war. First, we manually corrected apparent anomalies in the PTime field and used DTime to fill in missing PTime values. Then, we removed records with clearly abnormal formats or content. Finally, for duplicate news entries with identical sources and content, only one copy was retained to eliminate redundancy.

Source Country Identification

Method Description

We use the URL field of news articles and the large language model API provided by DeepSeek to perform intelligent analysis and structured extraction of news sources. For each article, we prompt the model to identify the country of origin and return structured JSON with country name, ISO code, and capital location. Regular expressions clean and deserialize this data into a CSV file.

Automatic News Summarization

Model Resource Link

https://huggingface.co/facebook/bart-large-cnn

Model Application

The model is loaded using Hugging Face's pipeline interface. After tokenizing the input text, if the token count is fewer than 100, the original text is retained; if it exceeds the predefined limit of MAX_TOKENS=1000, it is truncated to this length to avoid poor summary quality due to overly short input or decreased inference efficiency due to overly long input. During inference, the summary length is constrained to be no less than 100 tokens and no more than 250 tokens, ensuring the output remains within a reasonable range and is deterministic. In case of any exception, an error flag is returned. All generated summaries are written back to the Summary field of the dataset and exported as a CSV file.

Keyword extraction

Model Resource Link

https://github.com/LIAAD/yake

Model Application

First, we installed the YAKE library using pip install yake. During usage, we set the parameters as follows: the language was set to English (lan="en"), the maximum keyword length was limited to three-word phrases (n=3), the top 15 keywords were extracted (top=15), and the default deduplication method (such as seqm) was used to avoid redundant keywords. For each news article, the extracted results included keywords along with their importance scores (the lower the score, the more relevant the keyword). Additionally, we used YAKE's TextHighlighter tool to mark and highlight the keywords within the original text, making it easier for subsequent presentation and analysis.

Emotion Recognition

Model Resource Link

https://huggingface.co/j-hartmann/emotion-english-distilroberta-base

Model Application

Load the model through Hugging Face's pipeline interface and perform batch sentiment analysis on the news summary field. For each summary, the model outputs the corresponding multi-category sentiment scores. By sorting the scores, extract the top two sentiment labels and their confidence scores, and write them to the result file as Top1 and Top2 sentiment features, saved in UTF-8 encoding format as CSV.
